# QuantProject (全资产量化仓位决策系统)

## 📌 项目简介

这是一个自动化的多资产量化投资决策系统。系统通过固定的策略模型和技术指标（如移动均线、波动率偏离度等），对包含美股、日股、黄金和加密货币在内的核心资产进行资金分配与仓位建议。当前版本的重点是本地数据优先、按需增量同步和离线决策报告生成。
默认运行入口仍以目录内的 `START.bat` 为准。

## 🎯 核心资产及资金分配

系统目前的标准资金配置比例如下：
* **SPY (标普 500 ETF)**：21.25%
* **QQQ (纳斯达克 100 ETF)**：21.25%
* **EWJ (日本股市 ETF)**：21.25%
* **XAU (现货黄金)**：21.25%
* **BTC (比特币)**：15.00%

## 📂 核心文件架构

* **`START.bat`**：系统的主入口脚本。负责环境检查、自动创建目录以及一键依序运行「按需数据同步」和「引擎决策」环节。
* **`config.py`**：系统全局配置文件。配置项包含相对路径自动解析 (`BASE_DIR`)、金融平台代码映射（`Stooq` 和 `yfinance` 代码字典）以及上述的仓位权重配置。
* **`data_io.py`**：共享数据访问层。统一处理 CSV 标准化、SHA256 校验、列名识别和月度收盘价装载。
* **`strategies/`**：策略层。每个资产策略以独立类实现，通过注册表装配到引擎，便于扩展和单测。
* **`update_stooq_fast.py`**：多线程数据同步模块。启动时会先检查本地 CSV 是否已经足够新鲜；如果数据已覆盖最近交易日，则直接跳过网络请求。只有在确实需要刷新时，才会优先通过 `yfinance` 拉取最近窗口数据，并在失败时降级到 `Stooq` 备用接口。
* **`decision_engine.py`**：调度层。负责读取用户输入、装配策略、汇总报告并写入文本/JSONL 归档。

## 🧠 量化策略模型一览

1. **SPY (Sigmoid Smooth 策略)**
   使用 12 个月移动平均线 (MA12) 的偏离度作为输入，通过 Sigmoid 函数平滑计算权重。偏离度在 ±10% 时触发强多/空头规则。
2. **QQQ & EWJ (Combo Adaptive 策略)**
   计算多周期移动平均（MA3/6/9/12）结合短期波动率（Vol）计算出动量因子与波动率调整 Alpha，内建动量崩盘过滤机制防范暴跌风险。
3. **XAU (Trend Discrete 策略)**
   使用 MA6、MA9、MA12、MA15 进行多级别离散趋势判定，跌破均线会阶梯式减仓或清仓。
4. **BTC (Trend Fast MA6 策略)**
   采用快速均线反应（MA6），根据价格是否超过该均线判定是全仓进攻还是空仓防守。

## 🚀 运行方式

1. 确保安装了 Python，并在终端运行 `pip install pandas numpy requests yfinance` 补全环境依赖。
2. 双击运行（或在终端运行）目录下的 `START.bat` 文件。
3. 系统将自动执行：
   * 检查和创建 `data` 目录。
   * 先判断本地数据是否已经覆盖最近交易日，覆盖则直接跳过同步。
   * 在确实需要刷新时，按资产并发同步最近窗口的数据并自动合并到本地 CSV。
   * 运行决策引擎，如果终端提示，可以通过输入您的「实盘总资金」来进行精确的建仓金额规划（回车默认纯百分比显示）。
   * 控制台输出决策报告图表，并将结果持久化追加归档至 `position_history.txt` 记录文件。

## ⚙️ 常见问题 (FAQ)
* **启动还是慢怎么办？**：通常是首次运行、某个资产本地文件缺失，或者外部数据源临时超时。日常启动会优先走本地最新数据，只有在需要补齐最近交易日时才联网。
* **下载失败或网络波动怎么办？**：脚本会先尝试 `yfinance` 的增量窗口抓取，失败后自动降级到 `Stooq`。如果两个来源都不可用，先检查网络再重试。
* **可以增减交易资产吗？**：可以通过在 `config.py` 里的 `ASSETS_MAPPING` 和 `ALLOCATION_WEIGHTS` 字段进行调整，再在 `strategies/registry.py` 注册对应策略即可。
