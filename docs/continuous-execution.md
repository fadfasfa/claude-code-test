# 连续执行规则

本文件定义 Codex 在已接受的多步任务中如何继续执行，且不扩大范围。

## 可以继续

只有同时满足以下条件，才无需再次确认：

- 用户已经接受目标或方案。
- 下一步仍在已接受范围内。
- 写入路径位于选定工作区或允许的仓库治理文件内。
- 不需要安装、卸载或升级依赖。
- 不涉及全局配置、KB 仓库、凭据、auth、token、cookie、API key 或 proxy secret。
- 不需要高风险 Git 操作。
- 脏树归属清楚。
- 验证命令是受限的只读、status、test 或 smoke 命令。

## 必须停下确认

出现以下任一情况就停止并确认：

- 目标工作区不清楚。
- 任务范围改变。
- 同一文件混有无关用户改动和当前任务必须改动。
- 下一步需要安装依赖、修改全局配置、编辑 KB 仓库、触碰凭据路径或执行高风险 Git 操作。
- 任务会新增超过 3 个目标文件、跨入新模块、删除文件、改变业务目标或触碰破坏性数据操作。

## 临时记录

`.tmp/active-task/current.md` 可作为已忽略的长任务临时草稿。它不是规则来源、授权来源或提交产物。

## 完成

完成报告必须包含修改文件、验证命令与结果、剩余风险。无法验证时直接说明原因。
