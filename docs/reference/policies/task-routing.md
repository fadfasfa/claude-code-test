# 任务路由

本文件定义 `claudecode` 任务需要多少流程。

## 默认分级

| 规模 | 信号 | 流程 |
| :--- | :--- | :--- |
| Small | 窄文档编辑、单文件修复、错别字、本地配置澄清、无共享契约变化 | 确认范围，修改，运行最接近验证，报告 |
| Medium | 一个工作区内多文件、本地行为变化、测试入口不清、中等 UI/API 变化 | 简短计划，确认验证，小步修改，验证，报告 |
| Large | 跨工作区、共享数据契约、重大重构、发布工作、高回滚成本、需求不清 | 对齐范围，拆分任务，设检查点，谨慎验证 |

分级不清时先走更轻流程，有证据再升级。

## 目标工作区

如果用户点名工作区，先检查 `docs/workflows/work_area_registry.md` 中对应行，再列出该工作区顶层目录并继续缩小范围。

不要一开始就递归扫描大型工作区，除非任务确实需要。

## 计划

琐碎文本编辑、一行修复、窄文档清理或只读报告不需要正式计划。

中等任务、工作流文档、skills、tools、settings、视觉验证或数据契约变化使用简短计划。

大型任务、高风险清理、迁移、依赖变化或广泛行为变化使用详细计划。

## 测试

使用最窄且有意义的验证。测试强度随风险提高。

文档任务、探索读取、微小文案修改或没有有效本地测试入口的项目，不强行 TDD。

## Subagent

Subagent 是可选能力。只有任务边界清楚、并行不会制造协调风险时才使用。

适合：只读探索、测试发现、失败归因、审查、文件所有权不重叠的实现切片。

避免：关键路径阻塞、同文件并发修改、共享规则文件、协调成本高于收益的任务。

## 完成基线

所有非只读完成报告都需要：

- 修改文件。
- 验证命令或无法验证原因。
- 结果。
- 剩余风险。
