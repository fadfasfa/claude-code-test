# api-design

Use this skill when adding or changing API contracts.

## Rules

- Keep names and shapes consistent with existing project APIs.
- Make breaking changes explicit.
- Prefer simple, documented request and response shapes.
- Include validation and verification evidence with the change.
- Do not attach hooks, MCP-only dependencies, or forced Git workflow.
