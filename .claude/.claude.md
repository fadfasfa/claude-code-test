# 协议 0 — 契约写入（收到 [HANDOFF-WRITE] 时唯一第一动作）

> 未完成协议 0 直接执行任务 = HMAC 必定失败。
> 必须输出 `[STAGE: CONTRACT_WRITTEN]` 后才能进入协议 0.5。

1. 将 `[HANDOFF-WRITE]` 后的完整内容**原样**覆写至根目录 `agents.md`（严禁增删字段、保留占位符）
2. 验证 `workflow_id` 为实际值（不得含 `待填写` 或 `wf-<描述>`）
3. 初始化 `.ai_workflow/runtime_state.json`：

```powershell
$wfId = (Select-String -Path agents.md -Pattern 'workflow_id：\[([^\]]+)\]').Matches[0].Groups[1].Value
$state = @{
    schema_version        = "5.3"
    workflow_id           = $wfId
    execution_status      = "pending"
    audit_status          = "pending"
    merge_status          = "blocked"
    retry_count           = 0
    verification_result   = ""
    verification_artifact = ".ai_workflow/test_result.xml"
}
New-Item -Force -Path .ai_workflow -ItemType Directory | Out-Null
$state | ConvertTo-Json -Depth 5 | Set-Content .ai_workflow/runtime_state.json
```

4. `python .ai_workflow/verify_workspace.py --anchor-only`
5. 输出：`[STAGE: CONTRACT_WRITTEN] agents.md 已写入，runtime_state.json 已初始化，HMAC 锚定完成。`
6. 衔接协议 0.5

# 协议 2 — 分支隔离与执行日志

`Branch_Name` 已存在 → `[HALT: BRANCH_EXISTS]` 挂起；不存在 → `git checkout -b <Branch_Name>`

**每完成一个逻辑单元，强制：**

控制台输出：
```
[THOUGHT: <标题>] 修改 <文件>：<为什么这么改，选了什么方案，覆盖了哪个边界>
```

追加事件日志（禁止批量延迟）：
```powershell
$entry = @{
    ts            = (Get-Date -Format "yyyy-MM-ddTHH:mm:ss")
    step          = "<逻辑单元描述>"
    files_touched = @("<文件路径>")
    change_reason = "<修改原因>"
    risk_note     = "<边界风险，无则 none>"
    test_impact   = "<对测试的影响，无则 none>"
} | ConvertTo-Json -Compress
Add-Content -Path .ai_workflow/event_log.jsonl -Value $entry
```

写入失败 → `[WARN: EVENT_LOG_WRITE_FAILED]` 挂起，等待人类介入。

**发现技术债务时主动写入黄牌（无需等 Node C）：**
```powershell
$yc = Get-Content .ai_workflow/yellow_cards.json -Raw | ConvertFrom-Json
$key = "<相对路径>:<函数名或简短描述>"
$yc.cards.$key = @{issue="<问题>";workflow_id="<wf_id>";ts=(Get-Date -Format "yyyy-MM-ddTHH:mm:ss")}
$yc.current_count = ($yc.cards.PSObject.Properties | Measure-Object).Count
$yc | ConvertTo-Json -Depth 5 | Set-Content .ai_workflow/yellow_cards.json
```

`git add` 只能指定 Target_Files 中的文件，commit：`feat/fix: <描述> [wf-id: <workflow_id>]`

# 固定规则（契约中不重复，从这里读）

**agents.md 只读**：协议 0 覆写后任何节点不得修改，运行时状态写 runtime_state.json。

**写权限矩阵**：

| 节点 | 可写 |
| :--- | :--- |
| 执行节点（你） | `execution_status` / `verification_result` / `verification_artifact` / `event_log`（追加）/ `yellow_cards`（追加）/ `agents.md`（协议 0 一次） |
| Node C | `audit_status` / `merge_status` / `retry_count` |
| verify_workspace.py | `.contract_hash`（禁止其他节点触碰） |

**DACL 保护**：`Lock-Core.ps1` / `Unlock-Core.ps1` / `run_verification.ps1` MIC 高完整性；`.ai_workflow/audit_log.txt` Deny-Delete；`.git/hooks/` Deny-Delete。`agents.md` 与 `.ai_workflow/` 不在保护范围，可正常覆写。

**工作目录隔离**：所有文件操作必须限定在项目根目录（`$PWD`）内。严禁操作：
- `C:\Users\<任何用户名>\AppData\`
- 用户目录下 `Documents\` / `Desktop\` / `Downloads\`
- `C:\Program Files\` / `C:\Windows\`
- 任何不以当前项目根目录开头的绝对路径

**浏览器截图隔离**：Playwright / Selenium 必须指定 `--user-data-dir=.ai_workflow/chrome_tmp`。任务结束后清理：
```powershell
Remove-Item .ai_workflow/chrome_tmp -Recurse -Force -ErrorAction SilentlyContinue
```

**禁止破坏性 Git 操作**（违反 → `[HALT: UNAUTHORIZED_GIT]` 挂起）：
- `git reset --hard`
- `git checkout main` / `git checkout master`
- `git branch -D` 或删除任何非当前 Branch_Name 的分支
- `git clean -fd` / `git clean -fdx`

**范围控制**：`git add .` 禁止，只能指定具体文件。

**分支隔离**：只在 Branch_Name 分支操作，禁止直接改 main。

**破冰规约**：基建文件须人类执行 `Unlock-Core.ps1` → commit `--no-verify` → `Lock-Core.ps1`。

# 信号常量表

| 信号 | 含义 |
| :--- | :--- |
| `[STAGE: CONTRACT_WRITTEN]` | 契约写入锚定完成 |
| `[STAGE: HMAC_OK]` | 鉴权通过 |
| `[STAGE: EXECUTION_DONE]` | 执行完成，等待审计 |
| `[HALT: STATE_MISMATCH]` | 状态不符，终止 |
| `[HALT: HMAC_FAILURE]` | HMAC 失败，终止 |
| `[HALT: BRANCH_EXISTS]` | 分支已存在，挂起 |
| `[HALT: UNAUTHORIZED_GIT]` | 非法 git 操作，挂起 |
| `[GITNEXUS: BLAST_RADIUS_OK]` | 爆炸半径在范围内 |
| `[GITNEXUS: BLAST_RADIUS_OVERFLOW]` | 超出范围，挂起 |
| `[GITNEXUS: QUERY_FAILED]` | 查询失败，降级 |
| `[WARN: EVENT_LOG_WRITE_FAILED]` | 日志写入失败 |
| `[WARN: TEST_PATH_MISMATCH]` | 测试路径不符 |
