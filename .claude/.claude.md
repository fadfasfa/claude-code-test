# 协议 0 — 契约写入（收到 [HANDOFF-WRITE] 时唯一第一动作）

1. 将 `[HANDOFF-WRITE]` 后的完整内容**原样**覆写至根目录 `agents.md`（严禁增删字段）
2. 验证 `workflow_id` 为实际值（不得含 `待填写` 或 `wf-<描述>`）
3. 初始化 `.ai_workflow/runtime_state.json`：

```powershell
$wfId = (Select-String -Path agents.md -Pattern 'workflow_id：\[([^\]]+)\]').Matches[0].Groups[1].Value
$state = @{
    schema_version        = "5.3"
    workflow_id           = $wfId
    execution_status      = "pending"
    audit_status          = "pending"
    merge_status          = "blocked"
    retry_count           = 0
    verification_result   = ""
    verification_artifact = ".ai_workflow/test_result.xml"
}
New-Item -Force -Path .ai_workflow -ItemType Directory | Out-Null
$state | ConvertTo-Json -Depth 5 | Set-Content .ai_workflow/runtime_state.json
```

4. **DACL 锁定 agents.md**（写入后立即执行，不可跳过）：

```powershell
icacls agents.md /deny "*S-1-1-0:(W,D,DC)" | Out-Null
```

5. 输出：`[STAGE: CONTRACT_WRITTEN] agents.md 已写入并锁定，runtime_state.json 已初始化。`
6. 衔接协议 0.5

# 协议 4 — 完工验证与交权

> **副作用清理**：Verification_Command 运行时产生的 `__pycache__`、`*.pyc` 及非 Target_Files 的数据副产物，由 Node C 在审计打回/熔断时执行 `git clean -fd` 统一清除，执行节点不承担此职责。

```powershell
.\run_verification.ps1 -Caller executor
```

- exit 0 → 执行以下步骤后再输出 `[STAGE: EXECUTION_DONE]`，**主动退出进程**

  **① 更新 PROJECT.md**（若项目根目录存在该文件）：

  ```powershell
  $projectDoc = "PROJECT.md"
  if (Test-Path $projectDoc) {
      $wfId      = (Select-String -Path agents.md -Pattern 'workflow_id：\[([^\]]+)\]').Matches[0].Groups[1].Value
      $today     = Get-Date -Format "yyyy-MM-dd"
      $targets   = (Select-String -Path agents.md -Pattern '  - (.+)').Matches | ForEach-Object { $_.Groups[1].Value } | Join-String -Separator ", "
      $objective = (Select-String -Path agents.md -Pattern '任务目标：(.+)').Matches[0].Groups[1].Value
      $row = "| $today | $wfId | $objective | $targets |"

      # a. 追加变更记录（六节）
      $content = Get-Content $projectDoc -Raw
      $content = $content -replace '(\| :--- \| :--- \| :--- \| :--- \|)', "`$1`n$row"
      Set-Content $projectDoc $content -NoNewline

      # b. 轮换检查：超过 30 行移旧记录到 PROJECT_history.md
      $lines = (Select-String -Path $projectDoc -Pattern '^\| \d{4}-\d{2}-\d{2}').Count
      if ($lines -gt 30) {
          $allLines  = Get-Content $projectDoc
          $tableStart = ($allLines | Select-String '^\| 日期').LineNumber - 1
          $dataLines  = $allLines | Where-Object { $_ -match '^\| \d{4}-\d{2}-\d{2}' }
          $toArchive  = $dataLines | Select-Object -First 20
          $toKeep     = $dataLines | Select-Object -Skip 20
          # 写入归档
          $header = "# PROJECT 变更记录归档`n"
          Add-Content "PROJECT_history.md" ($header + ($toArchive -join "`n"))
          # 重建表格：保留表头 + 剩余行
          $newContent = $content -replace ('(' + [regex]::Escape(($toArchive -join "`n")) + "`n"), ""
          Set-Content $projectDoc $newContent -NoNewline
          Write-Host "[PROJECT.md: CHANGELOG_ROTATED] 已归档 20 行至 PROJECT_history.md"
      }

      # c. 若任务新增了文件（Target_Files 中有新路径），提示人工补充二三四节
      $newFiles = git diff --name-only --diff-filter=A HEAD
      if ($newFiles) {
          Write-Host "[PROJECT.md: NEW_FILES_DETECTED] 以下新文件未在文档中登记，请补充二/三/四节："
          $newFiles | ForEach-Object { Write-Host "  - $_" }
      }

      # d. 将 PROJECT.md 改动纳入当前分支 commit（避免脏工作区导致 Node C 打回）
      git add PROJECT.md
      if (Test-Path "PROJECT_history.md") { git add PROJECT_history.md }
      git commit -m "docs: update PROJECT.md [wf-id: $wfId]"
      Write-Host "[PROJECT.md: UPDATED]"
  } else {
      Write-Host "[PROJECT.md: NOT_FOUND] 跳过文档更新"
  }
  ```

  > **写权限边界**：执行节点只能追加六节（变更记录）和发出新文件提示，不得改写一/二/三/四节的现有内容。
- exit 非0，自修复 < 3 次 → `[THOUGHT: SELF-FIX N]`，先递增计数器，修复后确认 diff 在 Target_Files 内，重跑：
  ```powershell
  $state = Get-Content .ai_workflow/runtime_state.json -Raw | ConvertFrom-Json
  $state.retry_count += 1
  $state | ConvertTo-Json -Depth 5 | Set-Content .ai_workflow/runtime_state.json
  ```
- exit 非0，自修复 = 3 次 → `execution_status = "failed"`，输出完整堆栈，挂起

# 固定规则

**agents.md 物理只读**：协议 0 完成 DACL 锁定后，任何节点（包括你）在文件系统层面均无法写入，无需依赖提示词约束。
> **已知局限**：Windows NTFS 中文件所有者（Owner）天生持有 WRITE_DAC 特权，技术上可自行撤销 Deny 规则。此锁的威慑对象是 AI 幻觉与意外操作，而非同用户上下文的蓄意攻击——后者属于宿主机安全范畴，不在本工作流防护边界内。

**写权限矩阵**：

| 节点 | 可写 |
| :--- | :--- |
| 执行节点（你） | `execution_status` / `verification_result` / `verification_artifact` / `event_log`（追加）/ `yellow_cards`（追加）|
| Node C | `audit_status` / `merge_status` / `retry_count` |

**DACL 保护**：`Lock-Core.ps1` / `Unlock-Core.ps1` / `run_verification.ps1` MIC 高完整性；`agents.md` 协议 0 后 Deny-Write；`.ai_workflow/audit_log.txt` Deny-Delete；`.git/hooks/` Deny-Delete。

**工作目录隔离**：所有文件操作必须限定在项目根目录（`$PWD`）内。严禁操作：
- `C:\Users\<任何用户名>\AppData\`
- 用户目录下 `Documents\` / `Desktop\` / `Downloads\`
- `C:\Program Files\` / `C:\Windows\`
- 任何不以当前项目根目录开头的绝对路径

**浏览器截图隔离**：Playwright / Selenium 必须指定 `--user-data-dir=.ai_workflow/chrome_tmp`。任务结束后清理：
```powershell
Remove-Item .ai_workflow/chrome_tmp -Recurse -Force -ErrorAction SilentlyContinue
```

**禁止破坏性 Git 操作**（违反 → `[HALT: UNAUTHORIZED_GIT]` 挂起）：
- `git reset --hard`
- `git checkout main` / `git checkout master`
- `git branch -D` 或删除任何分支
- `git clean -fd` / `git clean -fdx`
- 直接覆写 `.ai_workflow/test_result.xml`

**范围控制**：`git add .` 禁止，只能指定具体文件。

**分支隔离**：只在 Branch_Name 分支操作，禁止直接改 main。

**破冰规约**：基建文件须人类执行 `Unlock-Core.ps1` → commit `--no-verify` → `Lock-Core.ps1`。

# 信号常量表

| 信号 | 含义 |
| :--- | :--- |
| `[STAGE: CONTRACT_WRITTEN]` | 契约写入并锁定完成 |
| `[STAGE: EXECUTION_DONE]` | 执行完成，等待审计 |
| `[HALT: STATE_MISMATCH]` | 状态不符，终止 |
| `[HALT: BRANCH_EXISTS]` | 分支已存在，挂起 |
| `[HALT: UNAUTHORIZED_GIT]` | 非法 git 操作，挂起 |
| `[GITNEXUS: BLAST_RADIUS_OK]` | 爆炸半径在范围内 |
| `[GITNEXUS: BLAST_RADIUS_OVERFLOW]` | 超出范围，挂起 |
| `[GITNEXUS: QUERY_FAILED]` | 查询失败，降级 |
| `[WARN: EVENT_LOG_WRITE_FAILED]` | 日志写入失败 |
| `[WARN: TEST_PATH_MISMATCH]` | 测试路径不符 |
| `[AUDIT-DENY: BRANCH_MISMATCH]` | 分支与契约不一致（Node C 发出）|
