# Core Protocols (核心执行协议)

**0. 契约写入模式 (Contract Write-in Mode)**
- 若启动时接收到包含 `[HANDOFF-WRITE]` 标头的指令，进入契约写入模式，优先于一切其他协议执行：
  1. 将指令中携带的完整契约内容**覆写**至根目录 `agents.md`（严格使用指令中的原始内容，禁止自行增删任何字段）；
  2. 覆写完成后，向控制台输出：`[STAGE: CONTRACT_WRITTEN] agents.md 已更新，进入状态机摄入。`；
  3. 随即自动衔接协议 1，以新写入的 `agents.md` 为准继续执行。
- 若启动时不含 `[HANDOFF-WRITE]` 标头，跳过本协议，直接执行协议 1。

**1. 共享状态机摄入与状态化握手 (State Machine Intake)**
- **启动后首要动作**：读取根目录的 `agents.md`。必须严格锚定该文件的【全局上下文状态】、【需求收敛锁状态】以及为你分配的【算力路由权重】。
- **强制握手反馈**：在执行具体的编码动作前，必须向控制台简要输出你对 `agents.md` 中指令及前后置条件的理解。这是向审计节点留下你的"语义宣称"，防止语义偏移。

**2. 最小权限网关与暂存区输出 (Staging Area Output)**
- 适配 Windows MIC 物理防线与防篡改钩子，你对核心源文件与基建处于限制写入状态（强行覆写将触发系统拦截与回滚）。
- 你的代码产出必须通过 `git add <target_files>` 写入 **Git 暂存区**，严格按照契约中"确定性执行清单"指定的目标文件列表执行，禁止全量 `git add`。物理缓冲区（`.ai_workflow/audit_buffer/`）已废弃，严禁读写。

**3. 零掩饰原则 (Zero-Suppression)**
- 遇到底层语法审查拦截时，严禁使用抑制错误（如 `try...pass`、加忽略注释）等手段。若无法修复，立即挂起任务并上报日志请求云端更新契约。

**4. 完工移交与交权铁律 (Mandatory Auto-Audit Handoff)**
- 当你完成 `agents.md` 分配的 Step 并将代码推入暂存区后，严禁进行下一轮画蛇添足的代码重构。
- 你必须将 `agents.md` 的状态修改为 `[待 Node C 终审]`。
- **自动巡航挂起**：在终端强制输出纯 ASCII 文本："[STAGE: EXECUTION_DONE] Staged to Git. Task suspended. System is ready for Node C (Qwen Roo Code) auto-cruise audit." 随后主动退出进程，移交控制权。
- **【C-6 豁免声明】**：格式为 `[STAGE: *]` 的纯 ASCII 状态信号属于机器间协议级输出，豁免 C-6 中文约束，必须保持原始格式以供下游节点解析。

**5. 令牌卫生 (Token Hygiene)**
- 读取日志优先使用管道符。严禁大段回显源码，并在关键验证后清理多余上下文。
