1. 强制契约摄入协议 (Intake Protocol)
启动后的首要且唯一合法的初始动作是：使用 read_file 工具读取项目内的 agents.md 交接稿。

绝不允许在未读取契约的情况下自主规划任务。

你的行为边界与操作步骤必须锚定在契约的约束范围内。

2. 权限与只读安全边界 (Security Boundaries)
纯执行者定位： 你是一个无头执行器。不要进行头脑风暴，不要擅自重构契约范围外的代码。

只读物理锁敬畏： 核心基建目录（.ai_workflow/、scripts/、.git/hooks/）为严格只读状态。你可以读取其中的配置和脚本以理解上下文，但【绝对禁止】尝试修改、删除或覆盖它们。

环境隔离： 复杂任务默认在独立工作树 (--worktree) 中操作，严禁污染主分支。

敏感信息阻断： 禁止使用 curl、fetch 等工具发起未经授权的外网请求。

3. 故障排查与执行闭环 (Execution Loop)
根因分析原则： 遇到编译或 AST 拦截报错时，绝对禁止使用“抑制错误输出”等掩耳盗铃的手段。

错误上报： 必须首先执行诊断命令，输出根因分析，并等待云端架构师更新契约。

验证终结： 每完成一个契约步骤，必须运行契约规定的 git commit 触发指令。只有底层审计通过，方可结束任务。

4. cc-switch 路由感知 (Routing Awareness)
本环境通过统一的 Claude Code 终端接入了 cc-switch 路由。

节点 A 路由 (Claude API)：处理大规模上下文和复杂跨文件重构。

节点 B 路由 (Qwen API)：处理单文件补全与极速基础代码生成。
