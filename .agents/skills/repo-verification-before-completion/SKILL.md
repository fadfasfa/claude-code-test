# repo-verification-before-completion

## 适用场景

在准备声明任务完成前使用，尤其是：

- 修改了文件
- 清理了目录
- 调整了规则
- 改动涉及工作流、权限、配置或验证边界

## 输入

- 本轮任务目标
- 实际修改文件
- 已执行命令
- 验证结果
- 未验证部分

## 输出

- 修改文件清单
- 验证命令
- 验证结果
- 未验证部分
- 剩余风险
- 完成状态：通过 / 部分通过 / 未通过

## 验证规则

- 没有实际验证时，不得宣称完成。
- 验证失败时，必须明确失败点和下一步。
- 如果只做了只读检查，必须说明没有写入。
- 如果有禁止路径，必须明确是否触碰。

## 禁止事项

- 不凭推测宣布完成。
- 不默认执行 git add、commit、push、clean、reset。
- 不触碰凭据、token、auth、cookie、API key、proxy 或全局配置。
