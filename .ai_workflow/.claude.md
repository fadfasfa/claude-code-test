【🌐 环境与极简三轴防线】
环境：Win11 (PowerShell) / VS Code。
本地项目已部署基于 HMAC-SHA256 与 AST 的安全网关，收敛为三个核心指令。你在执行修改前后，必须严格遵循以下护栏动作：
1. 拦截开启：`.ai_workflow/guard_on.py`（Qwen 零信任模式）。
2. 拦截关闭：`.ai_workflow/guard_off.py`（签发提权令牌，供修改基建使用）。
3. 自动审查与提交：`.ai_workflow/guard_audit.py <目标文件>`（修改任何源文件后，必须使用此脚本进行 AST 扫描、防降级校验与自动 Commit）。

【⚖️ 执行铁律 (Iron Rules)】
1. 强制语言：你的所有思考过程、执行计划、终端输出总结、代码注释，必须全程使用【简体中文】。
2. 零发散修改：严格按照交接稿（Contract）中指定的 `target` 和 `instructions` 修改代码。严禁波及目标模块之外的文件，严禁擅自“润色”未授权的逻辑。
3. 高危调用限制：除非交接稿明确豁免，否则严禁在代码中注入 `os.system`、`eval`、`exec` 等高危系统调用，严禁将精确的异常捕获（如 `except ValueError:`）降级拓宽（如 `except Exception:`）。

【🚦 标准操作流 (SOP)】
当你接收到 JSON 或 Markdown 格式的交接稿时：
1. 读取 Meta：确认 `executor_node`。若需修改基建且人类未授权，必须暂停并提示人类执行 `python .ai_workflow/guard_off.py`。
2. 物理落盘：将修改写入目标文件。
3. 护栏收尾：强制运行交接稿中的 `test_command`（通常为 `python .ai_workflow/guard_audit.py <修改的文件>`）。
